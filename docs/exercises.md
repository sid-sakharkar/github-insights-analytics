# Hands-on Coding Exercises and Mini-Projects for Learning Data Analysis with GitHub Data

This document contains 10 exercises and mini-projects designed to help you learn data analysis using GitHub data. Each exercise builds your skills in data manipulation, visualization, and understanding open-source contributions.

## Exercise 1: Repository Stars Analysis  
- **Objective**: Analyze the number of stars received by repositories over time.  
- **Data**: Use the GitHub API to extract data about repositories, their stars, and the timestamps.  
- **Task**: Create a line chart to visualize star growth for the top 10 repositories.

## Exercise 2: Contributor Activity  
- **Objective**: Measure and compare contributor activity across multiple repositories.  
- **Data**: Fetch data on commits for a set of repositories.  
- **Task**: Generate a bar chart showing the number of contributions per contributor.

## Exercise 3: Forks Comparison  
- **Objective**: Compare the number of forks for repositories over time.  
- **Data**: Gather data on forks through the GitHub API.  
- **Task**: Create a scatter plot comparing forks for selected repositories.

## Exercise 4: Issue Resolution Time  
- **Objective**: Analyze how quickly issues are being resolved in different repositories.  
- **Data**: Collect data on issues and their timestamps.  
- **Task**: Calculate the average resolution time and visualize it using a histogram.

## Exercise 5: Language Distribution in Repositories  
- **Objective**: Explore the programming languages used in various repositories.  
- **Data**: Use the GitHub API to pull language statistics.  
- **Task**: Create a pie chart showing the distribution of programming languages.

## Exercise 6: Commit Patterns  
- **Objective**: Identify patterns in commit activity by day of the week.  
- **Data**: Gather commit data over a period.  
- **Task**: Analyze the data and visualize it in a heatmap.

## Exercise 7: Pull Request Analysis  
- **Objective**: Assess the number of pull requests and their merging rates.  
- **Data**: Extract pull request data from selected repositories.  
- **Task**: Create a dashboard showing pull request statuses (merged, open, closed).

## Exercise 8: User Engagement Metrics  
- **Objective**: Measure user engagement metrics (comments, reviews) on pull requests.  
- **Data**: Use the GitHub API to gather comments and review data.  
- **Task**: Visualize user engagement through a stacked area chart.

## Exercise 9: Time to First Response  
- **Objective**: Analyze the time taken for maintainers to respond to issues.  
- **Data**: Collect data on issue creation and first response times.  
- **Task**: Calculate and display average response times for various repositories.

## Exercise 10: Trends in Contributions  
- **Objective**: Identify trends in contributions over the years.  
- **Data**: Aggregate contribution data year-wise.  
- **Task**: Create a line chart showing the trend of contributions over the years.

---  
This guide provides you with a series of hands-on exercises to enhance your data analysis skills using GitHub data. Good luck!
